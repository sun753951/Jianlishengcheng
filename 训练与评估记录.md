# 训练与评估记录

## 数据集

- 数据集名称：54k Resume dataset structured
- 来源链接：https://www.kaggle.com/datasets/suriyaganesh/resume-dataset-structured
- 下载方式：`kagglehub.dataset_download("suriyaganesh/resume-dataset-structured")`
- 下载缓存路径：`/Users/zyu/.cache/kagglehub/datasets/suriyaganesh/resume-dataset-structured/versions/2`
- 原始字段：`01_people.csv`、`03_education.csv`、`04_experience.csv`、`05_person_skills.csv`
- 处理结果：从岗位名、教育经历、工作经历和技能字段构造 800 条生成式训练样本。

## 本地训练

训练环境：

- 设备：Apple M5，16GB 内存
- Python：项目内 `.venv`
- 框架：Hugging Face Transformers、PyTorch
- 训练设备：MPS

已执行训练：

```sh
.venv/bin/python server/scripts/train_resume_model.py \
  --sample-limit 128 \
  --max-steps 120 \
  --max-length 384 \
  --output-dir server/artifacts/resume_chinese_model
```

训练结果：

- 基础模型：`uer/gpt2-chinese-cluecorpussmall`
- 样本数：128
- 步数：120
- 训练耗时：约 60 秒
- 最终训练损失：约 0.82
- 模型产物：`server/artifacts/resume_chinese_model`

## 评估结果

评估命令：

```sh
.venv/bin/python server/scripts/evaluate_resume_model.py \
  --model-dir server/artifacts/resume_chinese_model \
  --limit 5 \
  --max-new-tokens 120
```

评估现象：

- 平均生成耗时：约 2216ms。
- 模型学到了“姓名、求职岗位、个人摘要、教育经历、核心技能”等简历结构。
- 仍存在岗位名丢失、技能重复、标点重复和表达不稳定问题。
- 当前本地小样本微调模型不适合作为最终展示效果，应作为“公开数据集 + Transformers 本地训练闭环”的验证产物。

## 当前采用策略

后端保留真实 Transformer 模型加载能力，可通过以下方式启用：

```sh
RESUME_DISABLE_TRANSFORMERS=0 \
RESUME_MODEL_PATH=server/artifacts/resume_chinese_model \
.venv/bin/python -m uvicorn server.app:app --host 0.0.0.0 --port 8000
```

同时，后端增加了生成质量校验：如果模型输出过短、没有包含目标岗位、重复严重或结构不完整，会自动回退到规则模板生成。这样既能证明训练和模型加载流程存在，也能保证 HarmonyOS 应用端演示效果稳定。
