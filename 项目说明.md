## 题目原文

### 33. 基于自然语言生成的智能简历生成系统设计与实现

主要内容：用户通过网页端填写个人基础信息（姓名、学历等）、工作经历、项目经验等内容，系统利用基于 Transformer 的自然语言生成模型，根据不同岗位要求和简历模板，自动生成个性化简历文本；提供多种简历样式供用户选择，支持在线编辑、下载 PDF 格式简历；系统还具备简历评分功能，从结构、内容等方面给出优化建议。

具体要求：使用公开的简历文本数据集（可在https://www.kaggle.com/datasets搜索）；基于 Hugging Face Transformers 库；编程语言推荐 Python 和 ArkTS；生成简历的语法正确率达到 98% 以上，单份简历生成时间小于 2 秒。

## 项目实现对齐要求

- 前端使用 ArkTS / ArkUI 实现用户信息填写、模板选择、简历预览、在线编辑、评分建议和 PDF 下载入口。
- 自然语言生成能力应优先通过 Python 服务端实现，模型方案对齐 Hugging Face Transformers。
- 若阶段内无法完成真实 Transformer 模型推理，需要在交付文档中明确说明替代方案、限制和后续接入方式。
- 后续开发和答辩说明以 `../background/实践题目2.docx` 中第 33 题原文为准。

## 当前实现概览

- HarmonyOS 应用端：负责信息填写、模板选择、预览展示、在线编辑、评分结果展示和 PDF 导出入口。
- Python 服务端：提供简历生成、评分和 PDF 生成接口，目录为 `server/`。
- AI 生成：后端接入 Hugging Face Transformers，并提供规则模板降级方案，保证模型下载或硬件不足时仍可完成演示闭环。
- 数据集与训练：已使用 Kaggle 的 `54k Resume dataset structured` 构造轻量训练样本；本地训练脚本位于 `server/scripts/`，训练记录见 `训练与评估记录.md`。
- 简历模板：提供标准（经典墨绿）、技术（深蓝）、校园（紫色）、高级商务（金色）、极简灰白、黑金双栏共 6 套模板。各模板排版结构差异化：
  - **STANDARD（经典墨绿）**：经典上下结构，单列布局，带照片（72px 圆角矩形）
  - **TECHNICAL（深蓝）**：深蓝左侧栏（35%宽，含照片+技能胶囊+联系方式）+ 白色右侧内容区（摘要+经历），双列布局
  - **CAMPUS（紫色）**：时间轴风格，紫色竖线+圆点串联所有经历区块，圆形头像（64px）
  - **EXECUTIVE（金色）**：居中品牌头部（姓名居中，**无照片**），金色装饰线，2列网格卡片布局
  - **MINIMAL（极简灰白）**：全灰白极简纯文字，**无照片**，保留经典上下单列结构
  - **BLACK_GOLD（黑金）**：暗色双栏布局（左栏 PROFILE/CONTACT/EDUCATION，右栏 SIGNATURE PROJECT/EXPERIENCE/STRENGTHS），金色点缀，圆形头像
- 新增 `entry/src/main/ets/style/` 样式目录，内含：
  - `Colors.ets`：全局品牌色常量（靛蓝主色 `#4F46E5`）和中性色/背景/语义色/圆角定义
  - `TemplateThemes.ets`：6 套模板的 `TemplateVisualTheme` 主题配置接口，每套独立定义配色、字体大小、头像尺寸/圆角、有无照片 (`hasPhoto`) 等布局属性，通过 `getTemplateTheme()` 函数按模板类型获取
- StepPreview 页面已重构为每个模板对应独立的 HTML 构建方法（如 `buildTechnicalHtml` 负责深蓝主题+技能标签卡片、`buildBlackGoldHtml` 负责深色模式+金色点缀等），ArkUI Builder 侧同步改造为对应布局结构；并提取 `parseSkillTags`、`buildPhotoHtml`（支持尺寸/圆角参数）等通用工具函数，新增 `HtmlSection` 接口规范数据传递。
- PDF：通过后端生成 PDF 字节流，应用端调用导出接口并展示导出结果（含文件名、大小等状态信息；已修复 `result.byteLength` → `result.data.byteLength` 属性访问问题）。
- 文档：运行配置、Prompt、数据集候选、指标限制和隐私说明见 `系统运行配置与AI提示词说明.md`。
