## 题目原文

### 33. 基于自然语言生成的智能简历生成系统设计与实现

主要内容：用户通过网页端填写个人基础信息（姓名、学历等）、工作经历、项目经验等内容，系统利用基于 Transformer 的自然语言生成模型，根据不同岗位要求和简历模板，自动生成个性化简历文本；提供多种简历样式供用户选择，支持在线编辑、下载 PDF 格式简历；系统还具备简历评分功能，从结构、内容等方面给出优化建议。

具体要求：使用公开的简历文本数据集（可在https://www.kaggle.com/datasets搜索）；基于 Hugging Face Transformers 库；编程语言推荐 Python 和 ArkTS；生成简历的语法正确率达到 98% 以上，单份简历生成时间小于 2 秒。

## 项目实现对齐要求

- 前端使用 ArkTS / ArkUI 实现用户信息填写、模板选择、简历预览、在线编辑、评分建议和 PDF 下载入口。
- 自然语言生成能力应优先通过 Python 服务端实现，模型方案对齐 Hugging Face Transformers。
- 若阶段内无法完成真实 Transformer 模型推理，需要在交付文档中明确说明替代方案、限制和后续接入方式。
- 后续开发和答辩说明以 `../background/实践题目2.docx` 中第 33 题原文为准。

## 当前实现概览

- HarmonyOS 应用端：负责信息填写、模板选择、预览展示、在线编辑、评分结果展示和 PDF 导出入口。
- Python 服务端：提供简历生成、评分和 PDF 生成接口，目录为 `server/`。
- AI 生成：后端接入 Hugging Face Transformers，并提供规则模板降级方案，保证模型下载或硬件不足时仍可完成演示闭环。
- 数据集与训练：已使用 Kaggle 的 `54k Resume dataset structured` 构造轻量训练样本；本地训练脚本位于 `server/scripts/`。
- 简历模板：提供标准、技术、校园、高级商务、极简灰白、黑金双栏共 6 套模板。
- PDF：通过后端生成 PDF 字节流，应用端调用导出接口并展示导出结果。
- 文档：运行配置、Prompt、数据集候选、指标限制和隐私说明见 `系统运行配置与AI提示词说明.md`。
