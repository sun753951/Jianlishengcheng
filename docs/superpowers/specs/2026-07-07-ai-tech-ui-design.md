# AI 科技感首页 UI 优化设计

## 目标

将智能简历生成系统首页从朴素表单界面优化为更有 AI 产品展示感的科技渐变风格，同时保留现有核心流程：填写信息、选择模板、生成简历、在线编辑、评分建议和 PDF 导出。

## 范围

- 优化 `entry/src/main/ets/pages/Index.ets` 的视觉样式和页面层次。
- 不改变现有数据模型、生成逻辑、评分逻辑、照片选择逻辑和导出模拟逻辑。
- 不新增第三方依赖。
- 不把隐私信息写入日志、测试数据或界面调试信息。

## 视觉方向

- 采用浅蓝、靛蓝、紫色为主的科技感渐变风格。
- 顶部增加醒目的产品区，突出“智能简历生成系统”和“AI 生成 / 评分 / 导出”能力。
- 主按钮使用蓝紫主色，次按钮使用浅蓝底和深蓝文字。
- 卡片使用浅色渐变、细边框和轻阴影，提升层次感。
- 简历预览保持白底可读性，只增强标题、技能标签和模板高亮。

## 页面结构

- 保留当前单页 `Scroll` 工作流，避免引入页面跳转或步骤切换。
- 候选人信息、模板选择、模板预览、在线编辑、评分面板仍作为独立区域展示。
- 顶部 Hero 区放在表单前，作为第一屏视觉焦点。
- 错误、导出成功和空状态继续在原区域内展示，但改为更醒目的状态样式。

## ArkUI 实现约束

- 使用 ArkTS 明确类型，不使用 `any`。
- 页面方法命名保持 `lowerCamelCase`。
- 只做 UI 样式和局部 Builder 调整，不把服务逻辑塞入页面构建过程。
- 长文本输入和简历正文保持足够行高、对比度和滚动空间。
- 按钮、卡片和输入框尺寸使用 `vp` 语义的数值，兼顾手机屏幕。

## 验收标准

- 首页第一屏比当前版本更有科技感，能明显看出 AI 简历产品定位。
- 现有表单填写、模板选择、生成、编辑、评分、导出按钮仍可用。
- 简历预览正文保持清晰可读，不被装饰元素遮挡。
- 构建或可用静态检查执行后记录结果；如果本机缺少 DevEco CLI 或 SDK，需要说明原因。
