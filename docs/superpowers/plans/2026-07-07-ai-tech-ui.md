# AI 科技感首页 UI 优化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将智能简历生成系统首页优化为更有 AI 科技感的渐变视觉风格，同时保留现有功能流程。

**Architecture:** 采用单文件局部 UI 优化方式，集中修改 `entry/src/main/ets/pages/Index.ets` 的 Builder、颜色、卡片、按钮和页面背景。数据模型、简历生成服务、评分服务、照片选择和导出模拟逻辑保持不变。

**Tech Stack:** HarmonyOS Stage 模型、ArkTS、ArkUI、DevEco/Hvigor 构建。

## Global Constraints

- 不新增第三方依赖。
- 不改变现有数据模型、生成逻辑、评分逻辑、照片选择逻辑和导出模拟逻辑。
- 不把隐私信息写入日志、测试数据或界面调试信息。
- 使用 ArkTS 明确类型，不使用 `any`。
- 页面方法命名保持 `lowerCamelCase`。
- 长文本输入和简历正文保持足够行高、对比度和滚动空间。

### Task 1: 首页视觉样式升级

**Files:**
- Modify: `entry/src/main/ets/pages/Index.ets`

**Interfaces:**
- Consumes: 现有 `Index` 状态、`ResumeTemplateType`、`TemplateOption`、`ResumePreviewData`、`ResumePreviewSection`。
- Produces: 视觉增强后的首页，保留现有按钮点击、输入框、模板选择、评分和导出行为。

- [ ] **Step 1: 增加复用视觉 Builder 和按钮样式方法**

在 `Index` 结构体中加入科技感徽标、能力标签和交互卡片 Builder；增加主按钮、次按钮、卡片颜色等辅助样式方法。

- [ ] **Step 2: 优化顶部 Hero 和页面背景**

将 `build()` 顶部标题区替换为带渐变感、能力标签和状态说明的 Hero 区；页面背景从纯灰改为浅蓝紫背景。

- [ ] **Step 3: 优化表单、模板、预览、编辑、评分区域**

统一区域卡片边框、阴影、背景色；模板选择改成更明显的选中态；按钮改为蓝紫主色与浅蓝次色；评分面板增加科技感层次。

- [ ] **Step 4: 保持简历预览可读性**

只增强预览头部、标签、分割线和重点项目区域，不降低正文对比度，不增加遮挡。

- [ ] **Step 5: 运行构建或可用检查**

优先运行 `devecocli build --modules entry --build-mode debug`。如果本机没有 DevEco CLI，则运行可用的 `hvigorw` 构建命令；如果都不可用，记录原因。
